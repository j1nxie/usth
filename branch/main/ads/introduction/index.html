1. Introduction to Algorithms and Data Structures¶

1.1. Computer program¶

A computer program is a collection of instructions a computer can execute to perform a task.

1.2. Data structure¶

A data structure is a data organization, management and storage format that enables efficient access and computation.

1.3. Flowcharts¶

Flowcharts are used in designing and documenting simple processes or programs (see Computer program). They help with visualizing and understanding a process. A table of common symbols used in flowcharts can be seen here.

1.4. Pseudocode¶

Pseudocode is a high-level description of an algorithm (see Algorithm). It is usually concrete and easy for human comprehension. An example of pseudocode would be:

Listing 1.1 findMax(a)¶
1for i = 2 -> n do
2   if a[i] > Max then
3      Max = a[i]
4   end if
5end for
6return Max

1.5. Algorithm¶

An algorithm is a finite sequence of well-defined instructions to solve a specific problem or to perform a computation.

1.5.1. Properties¶

Good algorithms must possess these following properties.

1.5.1.1. Finiteness¶

Finiteness of an algorithm means it must always terminate after a finite number of steps. The stopping conditions of the algorithm should be assessed.

1.5.1.2. Definiteness¶

Each instruction of an algorithm should be defined precisely and concisely. Repetition should be avoided at all costs.

1.5.1.3. Input/Output¶

An algorithms may not have inputs, but it will have one or more outputs available when the algorithm terminates.

1.5.1.4. Effectiveness¶

All operations in an algorithm must be sufficiently basic that they can be done exactly as given and in finite time.

1.5.2. Evaluation criterias¶

There are a few criterias often used to evaluate an algorithm.

1.5.2.1. Correctness¶

Correctness evaluates whether an algorithm always produces the expected results for each given input.

1.5.2.2. Efficiency/Complexity¶

An algorithm should be optimized to have the lowest running time possible, also known as having the lowest time complexity. During implementation of the algorithm, as variables are declared and located in memory, an algorithm should also be optimized to have the lowest allocated memory possible, also known as having the lowest space complexity.

1.5.3. Categories¶

Algorithms can be divided into a few categories, depending on the method used to solve a certain problem.

1.5.3.1. Recursive¶

A simple recursive algorithm converts the main problem into smaller sub-problems, with certain base cases. It will solve the base cases directly, and it will recur (hence, “recursive”) with a simpler sub-problem.

The sub-problems may overlap. As a recursive algorithm will solve all sub-problems until they reach the base case, sub-problems may be solved more than once.

1.5.3.2. Divide and Conquer¶

Divide and conquer algorithms are an extension of the simple recursive algorithms (see Recursive). Given a problem to be solved, these algorithms will split it into several smaller problems. Afterwards, it will solve each problem recursively, and combine all the sub-solutions to produce a solution for the original problem.

Traditionally, an algorithm is considered a “divide and conquer” algorithm if it contains at least two recursive calls.

Some examples of divide and conquer algorithms would be quicksort and merge sort.

1.5.3.3. Dynamic Programming¶

A dynamic programming algorithm remembers past results and uses them to find new results. The algorithm will cut the problem into a set of simpler sub-problems. Afterwards, it will solve each sub-problem once, and store their solutions, which can be used in later stages.

The biggest difference between dynamic programming and divide and conquer (see Divide and Conquer) is that sub-problems generally don’t need to overlap.

An example of dynamic programming would be this implementation in finding the \(n^{th}\) number in the Fibonacci sequence:

Listing 1.2 Fibonacci¶
 1int fibo(n) {
 2   if ((n == 0) || (n == 1)) {
 3      return n;
 4   }
 5
 6   if (fibo(n) != 0) {
 7      return fibo(n);
 8   }
 9
10   return fibo(n - 1) + fibo(n + 2);
11}
